可以做，而且建议不要依赖文件 `mtime`。切换机器、clone、checkout 后文件修改时间不可靠，应该用 **英文源文件内容 hash + git commit/date 快照** 来判断中文 overlay 是否落后。

我建议在 `docs_i18n/` 里加一个 source-repo-only 的翻译状态层：

```text
docs_i18n/
  README.md
  tools/i18n_status.py
  zh-CN/
    i18n.status.yaml
    i18n.status.md
    ...
```

核心是 `docs_i18n/zh-CN/i18n.status.yaml`，例如：

```yaml
language: zh-CN
source_root: docs
overlay_root: docs_i18n/zh-CN
policy:
  source_identity: sha256
  fallback_when_missing: english_source
  exclude:
    - acceptance_reference/story_pack/**

entries:
  project/client_server_engineering_plan.md:
    source_path: docs/project/client_server_engineering_plan.md
    overlay_path: docs_i18n/zh-CN/project/client_server_engineering_plan.md
    source_sha256: "..."
    source_commit: "abc1234"
    source_changed_at: "2026-04-27T..."
    translated_at: "2026-04-27T..."
    translated_by: "codex"
    reviewed_at: null
    status: current
    notes: "初版中文对照已完成"
```

配套一个小工具放在 `docs_i18n/tools/`，不放进会同步到 MetaNC 的主工具链里：

```bash
python3 docs_i18n/tools/i18n_status.py check --lang zh-CN
python3 docs_i18n/tools/i18n_status.py report --lang zh-CN
python3 docs_i18n/tools/i18n_status.py mark-current --lang zh-CN docs/project/client_server_engineering_plan.md
```

状态判断规则：

- `missing`: 英文页存在，中文 overlay 不存在
- `untracked`: 中文 overlay 存在，但 manifest 没登记
- `stale`: 英文源当前 hash 和记录的 `source_sha256` 不一致
- `current`: 中文翻译已确认对应当前英文源 hash
- `needs-review`: 已翻译但还没人工确认
- `intentional-fallback`: 明确决定暂时回退英文，不算漏翻
- `orphaned`: 中文 overlay 对应的英文源已经不存在

这样流程会比较清楚：

1. 英文文档变更后，运行 `check` 能看到哪些中文页变 `stale`。
2. 翻译或确认后，运行 `mark-current` 更新那一页的 hash/commit/date。
3. `report` 生成 `i18n.status.md`，给人直接看进度。
4. 这些状态文件都在 `docs_i18n/` 下，已经被 MetaNC export/import 排除，不会污染 `MetaNC/nrt/hmi`。
5. 后续可以把 `check` 接到 `./tools/build_docs_html.sh` 前做 warning，或者只在 CI / agent handoff 时跑。

关键点是：**manifest 记录“翻译时对应的英文内容身份”，不是记录当前英文状态**。否则一变更就自动更新 hash，会把“中文已过期”这个事实盖掉。

我建议下一步直接实现最小版：`i18n.status.yaml` + `i18n_status.py check/report/mark-current` + 初始化当前 `zh-CN` overlay 状态。这样后续换机器、换人、换 agent，都能先跑一次 check 明确知道中文翻译债务在哪里。
