## 3. 本次关键决策摘要

### 决策 1: 不输出内部原始思维链

原因:

- 原始内部推理不适合作为工程审计材料
- 摘要化过程记录更适合阅读与复盘

替代方案:

- 输出结构化决策记录
- 输出工作流图和架构图

### 决策 2: 先恢复真实阻塞，再谈体验优化

原因:

- 缺失模块和坏脚本会让所有后续判断失真
- 先让 pipeline 工作，再处理一致性

### 决策 3: 一致性必须进入自动化测试

原因:

- 靠肉眼比较不可持续
- 没有测试的“一致性要求”很容易在后续迭代中丢失

### 决策 4: 去掉非语义噪声

原因:

- 本地路径不是 UI 语义的一部分
- 如果路径进入产物，就会制造伪差异

### 决策 5: 引入 fingerprint 而不是只依赖 session_id

原因:

- `session_id` 是人工标签，不是语义身份
- fingerprint 才能表达“这是不是同一份输入证据”

## 4. 对话中反复确认的原则

- 图片是 evidence，不是 retained truth
- 真正驱动 Web/QML 的是 retained DSL
- 同图一致性需要输入、导入、生成三层同时收紧
- 先保证功能正确，再追求视觉一致
- 一致性必须有自动化验证

## 5. 当前对你最重要的阅读结论

如果你后续要继续用这套工程做图驱动 HMI 生成，应该这样理解:

1. 图片不会直接生成最终 UI，它先进入 evidence 层
2. retained DSL 才是最终生成 Web/QML 的唯一稳定语义源
3. 现在这套工程已经开始具备“同语义输入稳定复现”的基础能力
4. 下一阶段真正决定体验上限的是 screenshot -> retained DSL 的语义归一化质量
