我建议改成 **列表优先，详情按需打开**。

现在右侧固定详情面板不太适合 Logs 页，因为日志页的核心动作是“扫描、过滤、比较多列信息”。右侧常驻会把表格压窄，而日志表本身又有 `time / level / source / category / event / message / context` 这些横向字段，所以很容易变成横向滚动和信息截断。

我推荐这一版：

1. **默认只显示全宽日志表**
   - Logs 页进入后不显示右侧详情。
   - 表格占满横向空间。
   - 当前行只做选中高亮。

2. **点击某行后，在底部打开详情抽屉**
   - 详情从表格下方展开，或者页面底部浮出。
   - 不占用横向空间。
   - 可以显示完整 JSON、request/session/correlation/page 等字段。
   - 右上角有关闭按钮。
   - 支持 Copy Entry / Copy JSON。

3. **表格列也做一次收敛**
   - 默认列保留：`Time / Level / Source / Event / Message`
   - `Category / Context` 可以放到详情里，或者 Context 只显示一个很短的 summary。
   - 这样日志页更像一个真正可扫读的运行日志，而不是字段堆满的数据库表。

4. **后续可加一个详情模式设置**
   - `Collapsed`：默认，不显示详情。
   - `Bottom`：选中后底部详情。
   - `Side`：高级用户可打开右侧详情。
   - 但第一版我不建议把设置做复杂，先默认 bottom detail 就够。

对应到现在实现，web 侧就是把 `log-view-body` 从双栏：

```css
grid-template-columns: minmax(0, 1fr) minmax(280px, 34%);
```

改成单栏，然后 `renderRuntimeLogDetail()` 不再常驻右侧，而是按 `selected_log_code` 和一个 `log_detail_open` 状态渲染到底部。QML 侧也同理，把右侧文本详情换成底部 panel 或 popup。

我最推荐：**全宽表格 + 点击行打开底部详情面板**。这是对现在结构改动不算大，但视觉收益最大的一版。
