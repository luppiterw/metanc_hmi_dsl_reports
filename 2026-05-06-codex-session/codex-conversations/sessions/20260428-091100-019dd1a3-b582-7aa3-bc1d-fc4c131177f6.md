### Codex Response 2 / final_answer
`2026-05-06 15:54:55 +0800`

```
`Run Retention` 是“手动执行日志保留策略”的按钮，不是清空日志。

当前实现里它会调用 server API：

`POST /api/runtime/logs/retention/run`

默认不传参数时，server 用这些保留周期：

- client `debug/info`：删除 3 天前的
- server `debug/info`：删除 7 天前的
- `warn/error/fatal`：删除 30 天前的
- `audit`：不会被这个流程删除

也就是说它主要是清理旧的低价值日志，避免 SQLite log store 一直膨胀。执行后 server 会追加一条 audit 日志：`logs.retention.run`，记录执行过 retention、删了多少、cutoff 时间是什么。Web 端会重新查询日志列表。

相关实现：
- UI 入口：[widget_emitters.py](/home/iaar/workspace/ccmix-wp/metanc_hmi_dsl/client/web_client/widget_emitters.py:2021)
- API 处理：[runtime_rest_controller.cpp](/home/iaar/workspace/ccmix-wp/metanc_hmi_dsl/server/src/transport/http/runtime_rest_controller.cpp:794)
- SQLite 删除策略：[sqlite_log_store.cpp](/home/iaar/workspace/ccmix-wp/metanc_hmi_dsl/server/src/logging/sqlite_log_store.cpp:426)

所以如果你只是想“把当前页面日志清爽一点”，`Run Retention` 不一定会马上删很多；它只删超过保留周期的 server-side 历史日志。当前还没有自动定时 retention schedule，这个按钮就是手动触发。
```
