# Conversation Report

## User Requests

- Confirm that PROG DIR cannot use the bottom `Return` key for folder-level back navigation.
- Add a browser-list `..` row for returning to the parent directory, while preventing root-level
  upward navigation.
- Confirm whether current directory creation is server-side.
- Fix `New File` so it creates inside the current program directory rather than root.
- Keep empty or one-entry folders visually consistent by preventing row-height stretching.
- Generate the daily report, update related docs, sync to MetaNC, then commit and push.

## Technical Decisions

- Keep backend resource entries clean. `res://program.browser.entries` contains real backend
  program entries only; the parent `..` row is generated by the Web/QML clients.
- Map the synthetic parent row to `progdir.commands.up` instead of inventing a fake file entry.
- Preserve the server-side root guard for `progdir.commands.up`, so UI and server both prevent
  navigation above the program workspace root.
- Keep `runtime_state.program_browser_selection` tied to real entry paths. The QML keyboard path
  treats the parent row as an action, not as a persisted selection.
- Scope `prog.commands.new` to the active browser directory on the server side, because strict
  server mode must not rely on the client to pre-join paths correctly.
- Fix the one-entry folder appearance in generated CSS rather than in generated HTML snapshots.

## Result

PROG DIR now separates page navigation from directory navigation. The footer `Return` key returns
to the program editor, while folder-level back navigation is available as a `..` row inside the
program browser only below root. New files and folders are created under the currently opened
program directory in strict server mode, and empty or sparse directories keep the same row geometry
as populated directories. The behavior is covered by regenerated Web/QML outputs, server tests,
mock runtime tests, updated bilingual docs, and a real split Web probe.
