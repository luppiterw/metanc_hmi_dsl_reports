server的strict和hybrid模式区别是什么

Codex Response 1 / commentary

2026-04-29 12:49:08 +0800

Codex Response 2 / final_answer

2026-04-29 12:49:58 +0800
`strict` 和 `hybrid` 是 client 端 runtime 怎么依赖 server 的模式，不是 server 自己的运行模式。

`strict`：真实分离模式。
Web/QML 不再用本地 seed 数据跑模拟，不启动本地执行 tick，属性/资源/命令都以 HMI Server 返回为准。启动后会走 `/bootstrap`，然后优先 WebSocket 订阅，失败再轮询 `/state`。JOG、CYCLE START、写资源、写属性都会发到 server。如果没配 server URL，会显示类似 `No HMI Server configured`，命令不会本地假执行。

`hybrid`：本地可独立跑的混合/调试模式。
client 会加载生成时带的 seed 数据，启动本地模拟 tick，命令先在 client 本地处理，所以没有 server 也能看界面变化、跑一些模拟逻辑。如果同时配置了 server URL，它会尝试 bootstrap 和 POST 命令/资源，但不会像 strict 一样完全依赖 server，也不会持续把 server 当唯一数据源。

简单说：

| 模式 | 没有 server | 数据来源 | 命令执行 |
|---|---|---|---|
| `strict` | 基本不可用 | server authoritative | 发给 server |
| `hybrid` | 可独立演示/调试 | 本地 seed + 本地模拟，可叠加 server 返回 | 本地先执行，也可顺带发 server |

现在真实 `run_split_web_native.sh 8010 8000` 这种分离运行，建议用 `strict`。
`hybrid` 更适合开发 Web/QML 页面、调样式、看静态生成结果，不适合作为判断真实 server 功能是否完成的依据。
